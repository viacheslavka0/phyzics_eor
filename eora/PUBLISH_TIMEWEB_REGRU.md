# Алгоритм: опубликовать ЭОР (ссылка для преподавателя)

**План:** VPS — **Timeweb Cloud**, домен — **REG.RU**. Так делать **нормально**: домен и сервер у разных компаний — стандартная схема. Важно только **в одном месте настроить DNS** (удобнее всего — в личном кабинете **REG.RU**, где лежит домен).

Полная техническая распаковка (Nginx, Gunicorn, Postgres, переменные Django) — в **`DEPLOY.md`**. Готовые шаблоны **`deploy/eora.env.example`**, **`deploy/nginx-site.conf`**, **`deploy/eora.service`** можно копировать на сервер и подправить пути/домен. Здесь — **порядок шагов от нуля до ссылки**.

---

## Этап 0. Уточнить у себя

- [ ] Есть доступ к **почте** (REG.RU и Timeweb пришлют доступы/подтверждения).
- [ ] На ПК умеете открыть **терминал** и зайти по **SSH** (логин/IP/пароль или ключ с панели Timeweb).
- [ ] В репозитории выполнен **`npm run build`** в `ui/`, статика лежит в `static/app/`, затем **`python manage.py collectstatic`** (каталог **`static_collected/`** для Nginx — см. **`DEPLOY.md`**).

---

## Этап 1. Домен на REG.RU

1. Зарегистрируйтесь / войдите на **reg.ru**.
2. Купите домен (любая зона, с которой согласны по цене: `.ru`, `.рф`, `.site` и т.д.).
3. **Пока VPS нет** — ничего страшного: DNS-записи добавите на этапе 3.

**Что учесть:** домен нужно **продлевать** каждый год (или срок, на который купили). Включите **автопродление**, если не хотите потерять имя.

---

## Этап 2. VPS на Timeweb Cloud

1. Создайте сервер (как вы уже смотрели в мастере):
   - **ОС:** Ubuntu 22.04 или 24.04.
   - **Регион:** Москва/СПб — если основная аудитория в РФ.
   - **Конфиг:** ориентир **2 GB RAM**, **30 GB NVMe**, **1 vCPU** (или 2 vCPU, если берёте запас).
   - **Публичный IPv4** — **нужен** (без него по ссылке из интернета не откроете).
   - **Бэкапы** — по желанию и бюджету (удобно для первого продакшена).
2. После создания в панели найдите **публичный IP** сервера (что-то вроде `185.xxx.xxx.xxx`).

---

## Этап 3. Привязать домен к IP (DNS на REG.RU)

Пока на сервере ещё «пусто», уже можно направить трафик на Timeweb.

1. В REG.RU: раздел **Домены** → ваш домен → **DNS** / **DNS-серверы и управление записями**.
2. Убедитесь, что домен использует **DNS REG.RU** (часто по умолчанию; если переключали на DNS Timeweb — тогда записи делайте там, где указаны DNS-серверы).
3. Добавьте запись типа **A**:
   - **Имя/поддомен:** `@` (корень) **или** `www` **или** `demo` (тогда сайт будет `https://demo.ваш-домен.ru`).
   - **Значение:** **IPv4 вашего VPS** с Timeweb.
   - TTL можно оставить авто или 300–3600.
4. Сохраните. Подождите **от 5 минут до нескольких часов** (распространение DNS).

**Проверка с ПК:**  
`ping demo.ваш-домен.ru` (или ваш хост) — должен показать **тот же IP**, что у VPS.

---

## Этап 4. Настроить сервер и приложение

Делайте по шагам из **`DEPLOY.md`** (разделы про Ubuntu, PostgreSQL, venv, `migrate`, Gunicorn, systemd, Nginx).

Кратко **порядок внутри сервера:**

1. `ssh` на сервер, обновления, `ufw` (22, 80, 443).
2. Установить **nginx**, **certbot**, **postgresql**, **python3-venv**, **git**.
3. Клонировать/скопировать проект в например `/srv/eora`.
4. Виртуальное окружение, `pip install -r requirements.txt`.
5. **Важно:** в `settings` для продакшена — **не** `DEBUG=True`, свой **`SECRET_KEY`**, **`ALLOWED_HOSTS`** с вашим FQDN (например `demo.ваш-домен.ru`), **PostgreSQL** вместо SQLite (см. `DEPLOY.md`).
6. `python manage.py migrate`, создать суперпользователя, тестовых учеников.
7. **Gunicorn** + **systemd** (слушает `127.0.0.1:8000`).
8. **Nginx:** прокси на Gunicorn + раздача **`/static/`** и **`/media/`** с диска (в `DEPLOY.md` есть пример).
9. **Certbot** `certbot --nginx -d demo.ваш-домен.ru` — получите **HTTPS**.

После этого ссылка для преподавателя:

`https://demo.ваш-домен.ru/app/`  
(или `https://ваш-домен.ru/app/` если запись **A** на `@`)

Админка: `https://.../admin/`

## Этап 5. Что ещё учесть (REG.RU + Timeweb раздельно)

| Тема | Комментарий |
|------|-------------|
| **Два кабинета** | Платите **REG.RU** (домен) и **Timeweb** (VPS) **отдельными счетами**; включите напоминания о продлении. |
| **DNS только в одном месте** | Записи **A** держите либо в REG.RU, либо на DNS Timeweb — **не дублируйте** противоречиво. |
| **Почта на домене** | Для демо ЭОР **не обязательна**. Позже можно добавить (REG.RU или отдельный сервис). |
| **Сертификат HTTPS** | Привязан к **имени**; после смены IP обновите запись **A** и при необходимости перевыпустите сертификат. |
| **Django `ALLOWED_HOSTS` и CSRF** | Должны совпадать с **точным** хостом из браузера (с `https`, без опечаток). |
| **Статика** | После каждого `npm run build` — снова залить/обновить файлы на сервере в `static/app/` (или ваш путь). |
| **Резервные копии** | БД + папка `media`; либо платные бэкапы Timeweb, либо свой `cron` + `pg_dump`. |

---

## Этап 6. Перед тем как отправить ссылку преподавателю

- [ ] Открывается **`https://.../app/`** без предупреждений сертификата.
- [ ] Вход **учеником** — пройден хотя бы один полный проход по сценарию.
- [ ] Вход **учителем** (`is_staff`) — открывается нужный интерфейс (через тот же `/app/`, см. `Portal.jsx`).
- [ ] Выдали **логин/пароль** тестовым пользователям (публичной регистрации нет — это нормально).

---

## Краткий ответ «нормально ли Timeweb + REG.RU»

**Да, нормально.** Часто так и делают: домен у регистратора с привычной панелью DNS, VPS — у облака с удобной ценой. Дополнительно покупать **отдельный «хостинг сайтов»** под этот проект **не нужно** — VPS и есть хостинг для Django.

Если что-то из шага 4 покажется непонятным — открывайте параллельно **`DEPLOY.md`** там, где уже расписаны команды и примеры конфигов.
