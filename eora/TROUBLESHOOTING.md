# Почему сервер не запускался? Разбор проблем

## Основные причины проблем

### 1. **Фоновый запуск через инструменты AI**

**Проблема:**
Когда я запускал сервер через `run_terminal_cmd` с флагом `is_background=true`, процесс мог:
- Запуститься, но не показать ошибки
- Завершиться сразу после запуска
- Неправильно работать с PowerShell

**Решение:**
✅ **Всегда запускайте сервер вручную** через командную строку или скрипт `START_SERVER.bat`

---

### 2. **Проблемы с URL маршрутизацией**

**Что было:**
- Использовались `lambda` функции в `urlpatterns` - они могут работать некорректно
- Использовался `FileResponse` с неправильными путями
- Неправильно настроены `TEMPLATES` в `settings.py`

**Что исправлено:**
- Используется `TemplateView` - стандартный Django подход
- Правильно настроен `TEMPLATES['DIRS']` на `BASE_DIR / "static" / "app"`
- Убраны lambda функции, используются обычные view функции

**Как проверить:**
```bash
python manage.py check  # Проверка на ошибки
python manage.py show_urls  # Показать все URL (если установлен django-extensions)
```

---

### 3. **Проблемы с путями к файлам**

**Проблема:**
- `Path(__file__).parent.parent` мог указывать не туда
- Относительные пути работали по-разному в разных контекстах

**Решение:**
- Используется `BASE_DIR` из `settings.py` - это надежнее
- Проверка существования файлов перед открытием

**Как проверить:**
```python
# В Python shell:
from pathlib import Path
from django.conf import settings
print(settings.BASE_DIR)
print((settings.BASE_DIR / "static" / "app" / "index.html").exists())
```

---

### 4. **Сервер запущен, но страницы не открываются**

**Возможные причины:**

#### a) Порт занят другим процессом
```bash
# Проверка:
netstat -ano | findstr :8000

# Решение: использовать другой порт
python manage.py runserver 8001
```

#### b) Неправильный URL
- Убедитесь, что используете `/app/` (со слешем в конце)
- Или `/teacher/` (со слешем в конце)

#### c) Кэш браузера
- Нажмите `Ctrl+F5` для жесткой перезагрузки
- Или `Ctrl+Shift+R`

#### d) Ошибки в консоли браузера
- Откройте DevTools (F12)
- Проверьте вкладку Console на ошибки JavaScript
- Проверьте вкладку Network - какие запросы не проходят

---

### 5. **Проблемы с виртуальным окружением**

**Проблема:**
- Виртуальное окружение не активировано
- Используется неправильная версия Python
- Не установлены зависимости

**Решение:**
```bash
# Активация:
.venv\Scripts\activate.ps1  # PowerShell
.venv\Scripts\activate.bat  # CMD

# Проверка:
python --version
pip list
```

---

### 6. **Ошибки в коде Django**

**Как проверить:**
```bash
# Проверка синтаксиса и настроек:
python manage.py check

# Проверка миграций:
python manage.py showmigrations

# Применение миграций (если нужно):
python manage.py migrate
```

**Типичные ошибки:**
- `ModuleNotFoundError` - не установлен пакет
- `ImportError` - неправильный импорт
- `TemplateDoesNotExist` - неправильный путь к шаблону
- `NoReverseMatch` - неправильный URL

## Как диагностировать проблемы

### Шаг 1: Проверьте, что сервер запущен
В окне сервера должно быть:
```
Starting development server at http://127.0.0.1:8000/
Quit the server with CTRL-BREAK.
```

Если этого нет - сервер не запустился. Смотрите ошибки выше.

### Шаг 2: Проверьте URL в браузере
- Правильный: `http://127.0.0.1:8000/app/`
- Неправильный: `http://127.0.0.1:8000/app` (без слеша)

### Шаг 3: Проверьте логи сервера
При запросе страницы в окне сервера должны появиться строки:
```
[22/Jan/2026 14:37:56] "GET /app/ HTTP/1.1" 200 932
```

Если видите `404` - проблема в URL маршрутизации.
Если видите `500` - ошибка в коде (смотрите traceback).

### Шаг 4: Проверьте консоль браузера
- F12 → Console - ошибки JavaScript
- F12 → Network - какие запросы не проходят

## Чек-лист перед запуском

- [ ] Виртуальное окружение активировано (если есть)
- [ ] Все зависимости установлены (`pip install -r requirements.txt`)
- [ ] Миграции применены (`python manage.py migrate`)
- [ ] Нет синтаксических ошибок (`python manage.py check`)
- [ ] Порт 8000 свободен
- [ ] Файлы `static/app/index.html` и `static/app/teacher.html` существуют
- [ ] Сервер запущен и показывает "Starting development server..."

## Рекомендации на будущее

1. **Всегда запускайте сервер вручную** - не через фоновые процессы
2. **Используйте скрипт `START_SERVER.bat`** - он делает всё автоматически
3. **Проверяйте логи сервера** - там видны все ошибки
4. **Используйте `python manage.py check`** перед запуском
5. **Проверяйте консоль браузера** при проблемах с интерфейсом

---

## Если ничего не помогает

1. Остановите сервер (Ctrl+C)
2. Выполните:
   ```bash
   python manage.py check
   python manage.py migrate
   ```
3. Запустите снова:
   ```bash
   python manage.py runserver
   ```
4. Откройте браузер в режиме инкогнито (чтобы исключить кэш)
5. Проверьте, что файлы существуют:
   ```bash
   dir static\app\index.html
   dir static\app\teacher.html
   ```

Если всё равно не работает - пришлите:
- Текст ошибки из окна сервера
- Скриншот консоли браузера (F12)
- Результат `python manage.py check`
