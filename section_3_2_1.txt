### 3.2.1. Модуль представления и осмысления системы знаний

Модуль представления и осмысления системы знаний реализует первый этап методики П. Я. Гальперина — этап создания ориентировочной основы действия. На этом этапе ученик знакомится с системой знаний, представленной в виде таблицы или схемы, и выполняет задания для проверки понимания основных понятий и связей между ними.

Представление системы знаний начинается с каталога, в котором ученик видит иерархическую структуру курса: классы, разделы, темы и системы знаний. Каталог реализован в виде раскрывающегося списка, где каждый уровень иерархии может быть развёрнут для просмотра вложенных элементов. При выборе системы знаний происходит загрузка её данных через API и создание или получение учебной сессии для отслеживания прогресса ученика.

После выбора системы знаний ученик попадает на этап осмысления. Интерфейс этого этапа разделён на две части: слева отображается изображение таблицы системы знаний с интерактивными зонами, справа располагаются вопросы для проверки понимания. Изображение загружается с сервера и масштабируется в зависимости от размера экрана, при этом координаты зон автоматически пересчитываются для корректного отображения поверх изображения.

Интерактивные зоны на изображении представляют собой прямоугольные области, которые учитель размечает при создании системы знаний. Каждая зона имеет координаты верхнего левого угла, ширину и высоту, а также метку для идентификации. При наведении курсора на зону она подсвечивается, а при клике на неё зона выделяется как выбранная. Это позволяет ученику выбирать зоны для ответов на вопросы типа «соответствие», где требуется указать, какие элементы таблицы относятся к определённому понятию.

Вопросы для осмысления системы знаний могут быть нескольких типов: открытый ответ, выбор одного варианта, множественный выбор и соответствие зон. Для вопросов типа «соответствие» ученик сначала активирует вопрос, а затем кликает на зоны таблицы, которые, по его мнению, являются правильным ответом. Выбранные зоны визуально выделяются, что позволяет ученику видеть свой выбор перед отправкой ответа на проверку.

Отдельный тип заданий — тексты с пропусками. Учитель создаёт текст, выделяет в нём слова или словосочетания, которые должны стать пропусками, и добавляет отвлекающие варианты для каждого пропуска. При отображении ученику текст показывается с выпадающими списками вместо пропущенных слов. Варианты ответов для каждого пропуска формируются автоматически: к правильному ответу добавляются отвлекающие варианты, и весь список перемешивается для исключения угадывания по порядку.

Проверка ответов выполняется на сервере при нажатии кнопки «Проверить ответы». Клиентская часть отправляет на сервер все ответы ученика: выбранные варианты для вопросов с выбором, текстовые ответы для открытых вопросов, выбранные зоны для вопросов типа «соответствие» и заполненные пропуски для текстов с пропусками. В листинге 3.7 показан фрагмент обработчика проверки ответов на клиентской стороне.

Листинг 3.7 – фрагмент обработчика проверки ответов этапа осмысления

```jsx
const handleCheck = async () => {
  setChecking(true);
  try {
    const mappings = ksData.questions
      ?.filter(q => q.type === "match")
      .map(q => ({
        question_id: q.id,
        selected_zone_ids: selectedZones[q.id] || []
      })) || [];

    const clozeAnswersArray = Object.entries(clozeAnswers).map(([key, value]) => ({
      gap_id: key,
      answer: value
    }));

    const res = await fetch(`/api/ks/${ksData.id}/check/`, {
      method: "POST",
      headers: {
        "Content-Type": "application/json",
        "X-CSRFToken": getCSRFCookie()
      },
      body: JSON.stringify({
        mappings,
        cloze_answers: clozeAnswersArray,
        answers
      })
    });

    const data = await res.json();
    setResults(data);
  } catch (e) {
    console.error(e);
    alert("Ошибка проверки: " + e.message);
  } finally {
    setChecking(false);
  }
};
```

Серверная часть проверяет каждый ответ ученика и сравнивает его с правильным ответом, хранящимся в базе данных. Для вопросов типа «соответствие» проверяется, совпадает ли набор выбранных зон с набором правильных зон. Для текстов с пропусками проверяется каждое заполненное слово на соответствие правильному ответу с учётом регистра. После проверки всех ответов вычисляется процент правильных ответов, и результат сравнивается с порогом успешности, который задаётся учителем при создании системы знаний (по умолчанию 85 процентов).

Если процент правильных ответов превышает порог успешности, этап осмысления считается пройденным, и ученик автоматически переходит к следующему этапу — формулировке типовой задачи. Если порог не достигнут, ученик видит результат проверки с указанием процента правильных ответов и может повторить попытку, исправив ошибки. Это соответствует методике Гальперина, где на этапе создания ориентировочной основы действия важно обеспечить полное понимание системы знаний перед переходом к следующему этапу.

Результат проверки сохраняется в базе данных в модели `ComprehensionAttempt`, которая связывается с учебной сессией ученика. Это позволяет отслеживать прогресс ученика и анализировать, какие элементы системы знаний вызывают затруднения. В листинге 3.8 показан фрагмент серверной логики проверки ответов.

Листинг 3.8 – фрагмент серверной логики проверки ответов

```python
@action(detail=True, methods=["post"])
def check(self, request, pk=None):
    ks = self.get_object()
    mappings = request.data.get("mappings", [])
    cloze_answers = request.data.get("cloze_answers", [])
    
    # Проверка соответствий (вопрос -> набор зон)
    q_map_correct = {}
    for q in KSQuestion.objects.filter(ks=ks).prefetch_related("correct_zones"):
        correct_ids = set(q.correct_zones.values_list("id", flat=True))
        chosen_ids = set()
        for item in mappings:
            if int(item.get("question_id", 0)) == q.id:
                chosen_ids = set(map(int, item.get("selected_zone_ids", [])))
                break
        q_map_correct[q.id] = (chosen_ids == correct_ids)
    
    # Проверка cloze
    cloze_correct = {}
    for cl in KSCloze.objects.filter(ks=ks):
        for blank in cl.blanks or []:
            key = f"{cl.id}:{blank.get('position', 0)}"
            correct_word = blank.get("correct", "").strip().lower()
            for ans in cloze_answers:
                if str(ans.get("gap_id", "")) == key:
                    answer = str(ans.get("answer", "")).strip().lower()
                    cloze_correct[key] = (answer == correct_word)
    
    # Подсчёт процента
    total_items = len(q_map_correct) + len(cloze_correct)
    correct_items = (
        sum(1 for ok in q_map_correct.values() if ok) +
        sum(1 for ok in cloze_correct.values() if ok)
    )
    score_percent = round((correct_items / total_items) * 100, 2) if total_items else 0.0
    passed = score_percent >= ks.comprehension_pass_threshold
    
    # Сохранение результата
    session.comprehension_score = score_percent
    session.comprehension_passed = passed
    if passed:
        session.current_stage = "typical_task"
    session.save()
    
    return Response({
        "passed": passed,
        "score_percent": score_percent,
        "mapping_feedback": [...],
        "cloze_feedback": [...]
    })
```

Таким образом, модуль осмысления системы знаний обеспечивает интерактивное знакомство ученика с учебным материалом и проверку понимания основных понятий и связей. Использование различных типов вопросов и интерактивных зон на изображении делает процесс осмысления более наглядным и вовлекающим, что способствует лучшему усвоению материала на этапе создания ориентировочной основы действия.
