### 3.2.2. Редактор схем «модели ситуации» для учителя

Редактор схем «модели ситуации» предназначен для создания эталонных графических представлений условий задач, которые в дальнейшем будут использоваться для сравнения с решениями учеников. Этот инструмент позволяет учителю визуализировать физическую ситуацию, описанную в задаче, используя стандартные графические элементы: точки, тела, векторы, линии координат и другие.

Редактор построен на библиотеке Konva.js, которая работает поверх холста HTML5 и предоставляет возможности для работы с векторной графикой. Это позволяет создавать масштабируемые изображения высокого качества, которые корректно отображаются на различных устройствах и могут быть экспортированы в растровые форматы для использования в других целях.

Интерфейс редактора разделён на несколько частей. Слева располагается палитра элементов, в которой элементы организованы по категориям для удобного поиска. Учитель может выбирать элементы из палитры и добавлять их на холст простым кликом. Также предусмотрена возможность создания собственных элементов схемы, если стандартного набора недостаточно. В центре находится рабочая область — холст, на котором строится схема. Справа для учителя доступна панель свойств, которая отображается при выборе элемента и позволяет изменять его параметры: координаты, размеры, цвет, поворот и другие свойства в зависимости от типа элемента.

Важной особенностью редактора является поддержка истории действий, которая позволяет учителю отменять и повторять операции. Это реализовано через механизм, который сохраняет состояние схемы после каждого значимого изменения. Учитель может отменить до 50 последних операций, используя стандартные сочетания клавиш или кнопки на панели инструментов. Это делает работу с редактором более удобной и позволяет экспериментировать с различными вариантами оформления схемы без страха потерять проделанную работу.

Для обеспечения точности построения схемы реализована функция привязки объектов к сетке. Когда привязка включена, элементы автоматически выравниваются по узлам сетки при перемещении, что обеспечивает аккуратное расположение объектов и упрощает создание симметричных и пропорциональных схем. Также поддерживается привязка объектов друг к другу: при перемещении элемента рядом с другим элементом появляется вспомогательная линия, показывающая, что объекты выровнены, и элемент автоматически «притягивается» к соседнему объекту. Это делает процесс построения схемы более интуитивным и помогает создавать аккуратные графические представления.

Редактор поддерживает множественное выделение элементов, что позволяет учителю работать сразу с несколькими объектами. Для выделения нескольких элементов используется сочетание клавиш Ctrl и клик по элементам, либо можно нарисовать рамку выделения, зажав левую кнопку мыши и переместив курсор. Выделенные элементы можно перемещать, удалять или дублировать одновременно, что значительно ускоряет процесс создания схем, особенно при необходимости размещения нескольких одинаковых объектов.

При работе с элементами доступно контекстное меню, которое появляется при нажатии правой кнопки мыши на элемент или группу элементов. Меню предоставляет быстрый доступ к основным операциям: дублированию, удалению, привязке к сетке и отмене последнего действия. Также все основные операции доступны через сочетания клавиш, что позволяет опытным пользователям работать более эффективно.

Для сохранения схемы реализован механизм, который отправляет данные схемы на сервер через API. Схема сохраняется в формате JSON, который содержит информацию о размерах холста и всех элементах с их свойствами. Это позволяет в дальнейшем загрузить схему для редактирования или использовать её для сравнения с решениями учеников. В листинге 3.9 показан фрагмент кода, демонстрирующий сохранение схемы учителем.

Листинг 3.9 – фрагмент кода сохранения схемы учителем

```jsx
const handleSave = async (data) => {
  setSaving(true);
  try {
    await api(`/api/teacher/tasks/${task.id}/save_schema/`, {
      method: "POST",
      body: JSON.stringify({
        data,
        template_type: "reference",
        name: `Эталонная схема: ${task.title}`,
      }),
    });
    alert("Схема сохранена!");
  } catch (e) {
    alert("Ошибка сохранения: " + e.message);
  } finally {
    setSaving(false);
  }
};
```

Схема сохраняется как шаблон типа «reference» (эталонная), что позволяет системе различать эталонные схемы, начальные заготовки и подсказки. Эталонная схема используется для автоматического сравнения с решениями учеников, если такая функциональность реализована, или для ручной проверки учителем.

Серверная часть получает данные схемы и сохраняет их в модели `SchemaTemplate`, которая связана с конкретной задачей. Это обеспечивает связь между задачей и её эталонным решением в виде графической схемы. В листинге 3.10 показан фрагмент серверной логики сохранения схемы.

Листинг 3.10 – фрагмент серверной логики сохранения схемы

```python
@action(detail=True, methods=["post"])
def save_schema(self, request, pk=None):
    task = self.get_object()
    schema_data = request.data.get("data", {})
    template_type = request.data.get("template_type", "reference")
    name = request.data.get("name", f"Схема для {task.title}")
    
    template, created = SchemaTemplate.objects.update_or_create(
        task=task,
        template_type=template_type,
        defaults={
            "name": name,
            "data": schema_data,
        }
    )
    
    return Response({
        "id": template.id,
        "created": created,
        "message": "Схема сохранена"
    })
```

Дополнительно редактор поддерживает автоматическое расширение холста при перемещении элементов близко к краю рабочей области. Это позволяет учителю создавать схемы любого размера, не ограничиваясь первоначальными размерами холста. Также реализована функция экспорта схемы в формат PNG, что позволяет сохранить изображение для использования в других целях, например, для включения в печатные материалы или презентации.

Таким образом, редактор схем для учителя представляет собой полнофункциональный инструмент для создания графических представлений физических ситуаций. Использование современных веб-технологий обеспечивает удобство работы и возможность создания качественных визуальных материалов без необходимости использования специализированного программного обеспечения.
