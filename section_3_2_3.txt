### 3.2.3. Интерфейс ученика по построению модели ситуации

Интерфейс ученика для построения модели ситуации реализован на основе того же редактора схем, который используется учителем, но с ограничениями функциональности, соответствующими целям обучения. Ученик использует редактор для создания графических представлений физических ситуаций, описанных в задачах, что является важным этапом в процессе решения задач.

Редактор схем для ученика доступен на этапах решения задач, где построение модели ситуации является необходимым шагом для понимания условий задачи и планирования решения. Интерфейс встроен в секцию «Модель ситуации», которая отображается перед полем для ввода ответа. Это позволяет ученику сначала визуализировать ситуацию, а затем перейти к вычислениям.

При открытии редактора для конкретной задачи система автоматически загружает начальную заготовку схемы, если она была создана учителем. Заготовка представляет собой шаблон типа «starter», который содержит базовые элементы схемы, помогающие ученику начать работу. Например, в заготовке могут быть уже размещены оси координат, основные объекты или ключевые точки ситуации. Это снижает когнитивную нагрузку на ученика и позволяет сосредоточиться на добавлении недостающих элементов, а не на создании схемы с нуля.

Если начальная заготовка отсутствует, ученик начинает работу с пустого холста, имея возможность создать схему самостоятельно. Это обеспечивает гибкость в использовании инструмента: для простых задач учитель может предоставить подробную заготовку, а для более сложных — дать ученику больше свободы в построении схемы.

Интерфейс редактора для ученика отличается от редактора учителя отсутствием панели свойств элементов. Это сделано для упрощения интерфейса и фокусировки внимания ученика на построении схемы, а не на тонкой настройке параметров элементов. Ученик может добавлять элементы из палитры, перемещать их на холсте, удалять и дублировать, но не может изменять свойства элементов через панель управления. Это соответствует дидактическим принципам, согласно которым ученик должен сосредоточиться на содержательной стороне задачи, а не на технических деталях инструмента.

Палитра элементов для ученика включает те же категории элементов, что и для учителя: точки и тела, векторы, линии и траектории, оси координат, надписи. Это обеспечивает единообразие в представлении схем и позволяет ученику создавать полноценные графические модели ситуаций. Элементы добавляются на холст простым кликом по элементу в палитре, после чего их можно перемещать и размещать в нужных позициях.

Редактор поддерживает основные операции работы с элементами: перемещение, удаление, дублирование. Для удаления элемента ученик может использовать контекстное меню, которое появляется при нажатии правой кнопки мыши, или клавишу Delete при выделенном элементе. Дублирование элементов доступно через контекстное меню или сочетание клавиш Ctrl+D, что позволяет быстро создавать несколько одинаковых объектов, например, несколько точек или векторов.

Система автоматически сохраняет схему ученика на сервер при нажатии кнопки «Сохранить». Сохранение происходит асинхронно, и во время процесса отображается индикатор загрузки, а после успешного сохранения появляется сообщение об успешном сохранении. Это позволяет ученику убедиться, что его работа сохранена, и в дальнейшем вернуться к ней или показать учителю для проверки.

Схема сохраняется в модели `StudentSchema`, которая связывает схему с конкретной задачей и пользователем. Это обеспечивает возможность отслеживания прогресса ученика и анализа его работы над построением моделей ситуаций. Сохранённые схемы могут использоваться для диагностики понимания учеником физических ситуаций и для предоставления обратной связи.

Интерфейс редактора для ученика имеет упрощённый вид по сравнению с редактором учителя. Отсутствует панель свойств справа, что делает интерфейс более чистым и понятным. Основное внимание уделяется рабочей области и палитре элементов, что способствует фокусировке на содержательной работе со схемой. Это соответствует принципам пользовательского интерфейса для образовательных приложений, где важно минимизировать отвлекающие факторы и помочь ученику сосредоточиться на учебной задаче.

Таким образом, интерфейс ученика для построения модели ситуации представляет собой адаптированную версию редактора схем, оптимизированную для учебных целей. Использование единой технологической базы для редакторов учителя и ученика обеспечивает согласованность в работе с схемами, а ограничение функциональности для ученика помогает сфокусировать внимание на содержательной стороне задачи.
