### 3.2.6. Интерфейс учителя для создания и управления контентом

Интерфейс учителя представляет собой комплексный инструмент для создания и управления всем учебным контентом электронного образовательного ресурса. Он обеспечивает учителю возможность создавать структуру курса, наполнять системы знаний учебными материалами, формировать задания для осмысления и задачи для решения, а также управлять графическими схемами. Интерфейс учителя реализован как отдельное веб-приложение, доступное по специальному маршруту и требующее авторизации с правами администратора.

Главная страница интерфейса учителя разделена на несколько основных разделов, доступ к которым осуществляется через навигационное меню в верхней части страницы. Основные разделы: «Структура курса», «Задачи» и «Элементы схем». Каждый раздел предоставляет специализированные инструменты для работы с соответствующим типом контента. Переключение между разделами происходит без перезагрузки страницы, что обеспечивает быструю навигацию и удобство работы.

Раздел «Структура курса» является центральным инструментом для организации учебного материала. Он отображает иерархическую структуру курса в виде трёх колонок: разделы, темы и системы знаний. Такое представление позволяет учителю видеть всю структуру курса одновременно и быстро находить нужные элементы. Каждая колонка содержит список соответствующих элементов с возможностью фильтрации по вышестоящим уровням иерархии. Учитель может создавать новые элементы, редактировать существующие и удалять их прямо из этого интерфейса.

При создании или редактировании раздела учитель указывает его название и порядковый номер для сортировки. При создании темы необходимо выбрать раздел, к которому она относится, указать название и порядковый номер. При создании системы знаний требуется выбрать тему, указать название, описание и статус публикации. Все изменения сохраняются на сервере через REST API, что обеспечивает синхронизацию данных между интерфейсом учителя и интерфейсом ученика.

Редактор системы знаний предоставляет учителю инструменты для наполнения системы знаний учебным контентом. Интерфейс редактора организован в виде вкладок, каждая из которых отвечает за определённый аспект системы знаний. Вкладка «Основное» позволяет редактировать название системы знаний, её описание, формулировку типовой задачи и статус публикации. Вкладка «Осмысление» предназначена для настройки этапа осмысления системы знаний: загрузки изображения таблицы, создания зон на изображении, добавления вопросов различных типов и текстов с пропусками.

Вкладка «Осмысление» разделена на две секции: «Изображение и зоны» и «Вопросы». В секции «Изображение и зоны» учитель может загрузить изображение таблицы системы знаний и создать на нём интерактивные зоны. Зоны создаются путём выделения прямоугольных областей на изображении с помощью мыши. Каждая зона имеет координаты верхнего левого угла, ширину и высоту, которые сохраняются в базе данных. Зоны используются в вопросах типа «соответствие», где ученик должен выбрать правильные элементы таблицы.

В секции «Вопросы» учитель может создавать вопросы различных типов: открытый ответ, выбор одного варианта, множественный выбор и соответствие зон. Для каждого вопроса указывается тип, текст вопроса, варианты ответов с отметкой о правильности и порядковый номер. Вопросы типа «соответствие» требуют дополнительной настройки: учитель должен указать, какие зоны на изображении являются правильным ответом. Также в этой секции учитель может создавать тексты с пропусками: вводить текст, выделять слова, которые должны стать пропусками, и добавлять отвлекающие варианты для каждого пропуска.

Вкладка «Метод решения» позволяет учителю создавать и редактировать метод решения задач для системы знаний. Учитель может добавить название метода, его описание и последовательность шагов. Каждый шаг содержит порядковый номер, название действия, подробное описание и подсказку для ученика. Метод решения структурируется как последовательность из десяти шагов, что соответствует методике П. Я. Гальперина.

Вкладка «Задачи» в редакторе системы знаний содержит ссылку на раздел «Задачи» в главном меню, где учитель может работать со всеми задачами системы. Это позволяет не дублировать функциональность редактирования задач в редакторе системы знаний, а использовать единый раздел для управления всеми задачами.

Раздел «Задачи» предоставляет учителю инструменты для создания и управления задачами всех систем знаний. Интерфейс содержит список всех задач с возможностью поиска по названию или тексту задачи и фильтрации по системе знаний. Для каждой задачи отображается её порядковый номер, название, краткий текст, уровень сложности, система знаний, к которой она относится, и наличие схемы. Учитель может создать новую задачу, отредактировать существующую, удалить задачу или перейти к редактированию схемы задачи.

При создании или редактировании задачи учитель заполняет форму, содержащую все необходимые поля: система знаний, порядковый номер, название, текст условия, правильный ответ (числовой или текстовый), единицу измерения, допустимую погрешность, уровень сложности и эталонные решения. Форма валидируется на клиентской стороне: проверяется наличие обязательных полей и корректность числовых значений. После сохранения задача становится доступной для учеников в интерфейсе решения задач.

Для каждой задачи учитель может создать графическую схему «модели ситуации», используя редактор схем, описанный в разделе 3.2.2. Редактор схем доступен из интерфейса редактирования задачи через кнопку «Схема». Созданная схема сохраняется как эталонная схема для задачи и используется для сравнения с решениями учеников. Также учитель может создать начальную заготовку схемы или подсказку для ученика.

Раздел «Элементы схем» позволяет учителю управлять библиотекой графических элементов, используемых в редакторе схем. Учитель может просматривать элементы, организованные по категориям, создавать новые категории и элементы, редактировать существующие элементы и удалять их. Каждый элемент содержит SVG-код для отображения, свойства по умолчанию и список редактируемых свойств. Это позволяет расширять библиотеку элементов и адаптировать её под специфические потребности учебного процесса.

Все операции в интерфейсе учителя выполняются через REST API, что обеспечивает разделение клиентской и серверной частей системы. API endpoints организованы по функциональным группам: для работы со структурой курса, системами знаний, задачами и элементами схем. Каждый запрос к API включает проверку прав доступа, что гарантирует, что только авторизованные учителя могут вносить изменения в учебный контент.

Интерфейс учителя реализован с использованием React и следует тем же принципам архитектуры, что и интерфейс ученика. Это обеспечивает единообразие в работе с системой и упрощает сопровождение кода. Все данные передаются в формате JSON, что позволяет легко интегрировать интерфейс учителя с другими компонентами системы.

Таким образом, интерфейс учителя представляет собой комплексный инструмент для создания и управления учебным контентом, который позволяет учителю самостоятельно наполнять электронный образовательный ресурс материалами без необходимости знания технических деталей работы системы. Это делает систему доступной для использования учителями-предметниками и способствует её практическому применению в образовательном процессе.
