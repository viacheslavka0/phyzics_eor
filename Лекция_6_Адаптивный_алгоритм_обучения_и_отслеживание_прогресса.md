## 1. Введение: что такое адаптивное обучение

### 1.1. Проблема традиционного обучения

В традиционных системах все ученики получают одинаковые задания:

```
Ученик 1 (сильный) → Задача 1 → Задача 2 → Задача 3
Ученик 2 (слабый)  → Задача 1 → Задача 2 → Задача 3
```

**Проблемы:**
- ❌ Сильным ученикам скучно (слишком легко)
- ❌ Слабым ученикам сложно (слишком трудно)
- ❌ Нет персонализации
- ❌ Низкая эффективность обучения

### 1.2. Решение: адаптивное обучение

**Адаптивное обучение** — система, которая подстраивает сложность заданий под уровень ученика.

**Принципы:**
- ✅ Сложность заданий соответствует уровню ученика
- ✅ Система реагирует на успехи и неудачи
- ✅ Персонализированный путь обучения
- ✅ Повышение эффективности обучения

**Пример:**
```
Ученик 1 (сильный):
  Задача 1 (легкая) ✓ → Задача 2 (средняя) ✓ → Задача 3 (сложная) ✓
  
Ученик 2 (слабый):
  Задача 1 (легкая) ✗ → Задача 1 (легкая) ✓ → Задача 2 (легкая) ✓
```

### 1.3. Типы адаптивных алгоритмов

**1. Правило "лестницы" (Staircase method):**
- При правильном ответе → увеличиваем сложность
- При неправильном ответе → уменьшаем сложность

**2. На основе процента правильных ответов:**
- Если > 80% правильных → увеличиваем сложность
- Если < 60% правильных → уменьшаем сложность

**3. На основе последовательности ошибок:**
- Если 2 ошибки подряд → уменьшаем сложность
- Если 3 правильных подряд → увеличиваем сложность

**4. Выбор ученика:**
- Ученик сам выбирает уровень сложности
- Система корректирует на основе результатов

## 2. Модели для отслеживания прогресса

### 2.1. Модель LearningSession

**Назначение:** Отслеживает прогресс ученика в рамках одной системы знаний.

```python
class LearningSession(models.Model):
    """Сессия обучения ученика"""
    user = models.ForeignKey(User, on_delete=models.CASCADE)
    ks = models.ForeignKey(KnowledgeSystem, on_delete=models.CASCADE)
    
    # Текущий этап обучения
    current_stage = models.CharField(
        max_length=50,
        default="comprehension",
        help_text="Текущий этап: comprehension, typical_task, solving, completed"
    )
    
    # Выбор сложности учеником
    difficulty_choice = models.CharField(
        max_length=10,
        choices=[("easy", "Лёгкое"), ("medium", "Непростое"), ("hard", "Трудное")],
        blank=True,
        help_text="Выбор ученика: насколько трудным показалось задание"
    )
    
    # Статистика решения задач
    tasks_solved_count = models.PositiveIntegerField(default=0)
    tasks_correct_count = models.PositiveIntegerField(default=0)
    wrong_attempts_in_row = models.PositiveIntegerField(
        default=0,
        help_text="Количество неправильных ответов подряд"
    )
    
    # Результаты осмысления
    comprehension_score = models.FloatField(default=0)
    comprehension_passed = models.BooleanField(default=False)
    
    # Итоговый результат
    score_percent = models.FloatField(default=0)
    passed = models.BooleanField(default=False)
    
    # Время
    started_at = models.DateTimeField(auto_now_add=True)
    finished_at = models.DateTimeField(null=True, blank=True)
```

**Поля для адаптации:**
- `difficulty_choice` — выбор ученика
- `wrong_attempts_in_row` — счётчик ошибок подряд
- `tasks_correct_count` / `tasks_solved_count` — процент правильных ответов

## 3. Алгоритмы адаптивного выбора заданий

### 3.1. Алгоритм на основе выбора ученика

**Принцип:** Ученик сам выбирает уровень сложности, система корректирует на основе результатов.

**Реализация:**
```python
def set_difficulty(session, difficulty_choice):
    """Установить уровень сложности, выбранный учеником"""
    session.difficulty_choice = difficulty_choice
    
    # Определяем следующий этап
    stage_map = {
        "easy": "solving_easy",
        "medium": "method_composition",
        "hard": "method_composition",
    }
    session.current_stage = stage_map[difficulty_choice]
    session.save()
```

**Преимущества:**
- ✅ Ученик чувствует контроль
- ✅ Простая реализация
- ✅ Учитывает самооценку ученика

**Недостатки:**
- ❌ Ученик может недооценить или переоценить себя
- ❌ Нужна дополнительная корректировка

### 3.2. Алгоритм на основе последовательности ошибок

**Принцип:** Если ученик делает ошибки подряд, уменьшаем сложность.

**Реализация:**
```python
def submit_task_answer(session, task, answer, is_correct):
    """Обработка ответа на задачу"""
    # Сохраняем попытку
    TaskAttempt.objects.create(
        session=session,
        task=task,
        answer_numeric=answer,
        is_correct=is_correct,
    )
    
    # Обновляем статистику
    session.tasks_solved_count += 1
    
    if is_correct:
        session.tasks_correct_count += 1
        session.wrong_attempts_in_row = 0  # Сбрасываем счётчик ошибок
    else:
        session.wrong_attempts_in_row += 1
        
        # Адаптивная логика: если 2 ошибки подряд в режиме "легко"
        # → переключаемся на "непростое"
        if session.difficulty_choice == "easy" and session.wrong_attempts_in_row >= 2:
            session.current_stage = "method_composition"
            session.difficulty_choice = "medium"
    
    session.save()
```

**Логика:**
- Правильный ответ → сбрасываем счётчик ошибок
- Неправильный ответ → увеличиваем счётчик
- 2 ошибки подряд в "easy" → переключаем на "medium"

### 3.3. Алгоритм "лестницы" (Staircase)

**Принцип:** При правильном ответе увеличиваем сложность, при неправильном — уменьшаем.

**Реализация:**
```python
def get_next_task_difficulty(session):
    """Определить сложность следующей задачи по алгоритму лестницы"""
    # Получаем последние попытки
    recent_attempts = TaskAttempt.objects.filter(
        session=session
    ).order_by('-created_at')[:3]
    
    if not recent_attempts:
        # Первая задача — средняя сложность
        return 3
    
    last_attempt = recent_attempts[0]
    current_difficulty = last_attempt.task.difficulty
    
    if last_attempt.is_correct:
        # Правильный ответ → увеличиваем сложность (но не больше 5)
        return min(current_difficulty + 1, 5)
    else:
        # Неправильный ответ → уменьшаем сложность (но не меньше 1)
        return max(current_difficulty - 1, 1)
```

**Пример работы:**
```
Задача 1 (сложность 3) → Правильно → Задача 2 (сложность 4)
Задача 2 (сложность 4) → Неправильно → Задача 3 (сложность 3)
Задача 3 (сложность 3) → Правильно → Задача 4 (сложность 4)
```

### 3.4. Алгоритм на основе процента правильных ответов

**Принцип:** Анализируем процент правильных ответов и корректируем сложность.

**Реализация:**
```python
def get_success_rate(session):
    """Вычислить процент правильных ответов"""
    if session.tasks_solved_count == 0:
        return 0.0
    return session.tasks_correct_count / session.tasks_solved_count

def adjust_difficulty(session):
    """Скорректировать сложность на основе успешности"""
    success_rate = get_success_rate(session)
    
    if success_rate >= 0.8:
        # Высокий процент правильных → увеличиваем сложность
        if session.difficulty_choice == "easy":
            session.difficulty_choice = "medium"
        elif session.difficulty_choice == "medium":
            session.difficulty_choice = "hard"
    elif success_rate < 0.6:
        # Низкий процент правильных → уменьшаем сложность
        if session.difficulty_choice == "hard":
            session.difficulty_choice = "medium"
        elif session.difficulty_choice == "medium":
            session.difficulty_choice = "easy"
    
    session.save()
```

**Пороги:**
- ≥ 80% правильных → увеличиваем сложность
- < 60% правильных → уменьшаем сложность
- 60-80% → оставляем как есть

### 3.5. Комбинированный алгоритм

**Принцип:** Комбинируем несколько подходов для более точной адаптации.

**Реализация:**
```python
def get_adaptive_difficulty(session):
    """Комбинированный алгоритм адаптации"""
    # Фактор 1: Процент правильных ответов
    success_rate = get_success_rate(session)
    
    # Фактор 2: Последовательность ошибок
    wrong_in_row = session.wrong_attempts_in_row
    
    # Фактор 3: Выбор ученика
    user_choice = session.difficulty_choice
    
    # Начальная сложность на основе выбора ученика
    if user_choice == "easy":
        base_difficulty = 2
    elif user_choice == "medium":
        base_difficulty = 3
    else:  # hard
        base_difficulty = 4
    
    # Корректировка на основе успешности
    if success_rate >= 0.8 and wrong_in_row == 0:
        # Высокая успешность, нет ошибок → увеличиваем
        base_difficulty = min(base_difficulty + 1, 5)
    elif success_rate < 0.6 or wrong_in_row >= 2:
        # Низкая успешность или ошибки подряд → уменьшаем
        base_difficulty = max(base_difficulty - 1, 1)
    
    return base_difficulty
```

## 4. Реализация адаптивной логики

### 4.1. Обработка ответа на задачу

**Полный пример из реального проекта:**

```python
@action(detail=True, methods=["post"])
def submit(self, request, pk=None):
    """Отправить ответ на задачу"""
    task = Task.objects.get(pk=pk)
    session = LearningSession.objects.get(
        pk=request.data.get("session_id"),
        user=request.user
    )
    
    answer_numeric = request.data.get("answer_numeric")
    answer_text = request.data.get("answer_text", "")
    
    # Проверка ответа
    is_correct = task.check_answer(answer_numeric or answer_text)
    
    # Сохраняем попытку
    attempt = TaskAttempt.objects.create(
        session=session,
        task=task,
        answer_numeric=answer_numeric,
        answer_text=answer_text,
        is_correct=is_correct,
    )
    
    # Обновляем статистику сессии
    session.tasks_solved_count += 1
    
    if is_correct:
        session.tasks_correct_count += 1
        session.wrong_attempts_in_row = 0  # Сбрасываем счётчик
    else:
        session.wrong_attempts_in_row += 1
        
        # Адаптивная логика: если 2 ошибки подряд в режиме "легко"
        # → переключаемся на "непростое"
        if session.difficulty_choice == "easy" and session.wrong_attempts_in_row >= 2:
            session.current_stage = "method_composition"
            session.difficulty_choice = "medium"
    
    # Вычисляем процент правильных ответов
    if session.tasks_solved_count > 0:
        session.score_percent = (
            session.tasks_correct_count / session.tasks_solved_count * 100
        )
    
    session.save()
    
    # Логируем событие
    EventLog.objects.create(
        user=request.user,
        session=session,
        event="task_submit",
        payload={
            "task_id": task.id,
            "is_correct": is_correct,
            "wrong_attempts_in_row": session.wrong_attempts_in_row,
        }
    )
    
    return Response({
        "is_correct": is_correct,
        "tasks_solved_count": session.tasks_solved_count,
        "tasks_correct_count": session.tasks_correct_count,
        "wrong_attempts_in_row": session.wrong_attempts_in_row,
        "current_stage": session.current_stage,
    })
```

### 4.2. Выбор следующей задачи

**Реализация:**

```python
def get_next_task(session):
    """Получить следующую задачу на основе адаптивного алгоритма"""
    ks = session.ks
    
    # Определяем желаемую сложность
    if session.difficulty_choice == "easy":
        target_difficulty = [1, 2]  # Лёгкие задачи
    elif session.difficulty_choice == "medium":
        target_difficulty = [2, 3, 4]  # Средние задачи
    else:  # hard
        target_difficulty = [4, 5]  # Сложные задачи
    
    # Получаем уже решённые задачи
    solved_task_ids = TaskAttempt.objects.filter(
        session=session
    ).values_list('task_id', flat=True)
    
    # Ищем задачу подходящей сложности, которую ещё не решали
    next_task = Task.objects.filter(
        ks=ks,
        difficulty__in=target_difficulty
    ).exclude(id__in=solved_task_ids).first()
    
    # Если все задачи решены, возвращаем любую
    if not next_task:
        next_task = Task.objects.filter(ks=ks).exclude(
            id__in=solved_task_ids
        ).first()
    
    return next_task
```

### 4.3. Переход между этапами

**Реализация:**

```python
@action(detail=True, methods=["post"])
def advance_stage(self, request, pk=None):
    """Перейти к следующему этапу"""
    session = LearningSession.objects.get(pk=pk, user=request.user)
    
    next_stage = request.data.get("next_stage")
    
    # Определяем следующий этап на основе текущего
    stage_flow = {
        "comprehension": "typical_task",
        "typical_task": "task_list",
        "task_list": "difficulty_assessment",
        "difficulty_assessment": "solving_easy",  # или method_composition
        "solving_easy": "method_composition",
        "method_composition": "step_by_step",
        "step_by_step": "completed",
    }
    
    if not next_stage:
        next_stage = stage_flow.get(session.current_stage)
    
    if next_stage:
        session.current_stage = next_stage
        
        # Если переходим на этап "completed", завершаем сессию
        if next_stage == "completed" and not session.finished_at:
            session.finished_at = timezone.now()
            session.passed = session.score_percent >= 70  # Порог прохождения
        
        session.save()
    
    return Response({
        "ok": True,
        "current_stage": session.current_stage,
    })
```

## 5. Аналитика и метрики

### 5.1. Основные метрики

**Метрики для ученика:**
- Процент правильных ответов
- Количество решённых задач
- Время на решение
- Количество ошибок подряд
- Прогресс по этапам

**Метрики для учителя:**
- Средний процент правильных ответов по классу
- Самые сложные задачи
- Среднее время решения
- Процент завершивших систему знаний

### 5.2. Вычисление метрик

```python
def calculate_session_metrics(session):
    """Вычислить метрики сессии"""
    attempts = TaskAttempt.objects.filter(session=session)
    
    metrics = {
        "total_tasks": session.tasks_solved_count,
        "correct_tasks": session.tasks_correct_count,
        "success_rate": session.score_percent,
        "average_time": 0,
        "wrong_in_row_max": session.wrong_attempts_in_row,
    }
    
    # Среднее время решения
    if attempts.exists():
        total_time = sum(a.time_spent_seconds for a in attempts)
        metrics["average_time"] = total_time / attempts.count()
    
    return metrics

def calculate_class_metrics(ks, school_class):
    """Вычислить метрики по классу"""
    sessions = LearningSession.objects.filter(
        ks=ks,
        user__schoolclass=school_class,
        finished_at__isnull=False
    )
    
    if not sessions.exists():
        return None
    
    total_sessions = sessions.count()
    passed_sessions = sessions.filter(passed=True).count()
    
    total_tasks = sum(s.tasks_solved_count for s in sessions)
    total_correct = sum(s.tasks_correct_count for s in sessions)
    
    return {
        "total_students": total_sessions,
        "passed_students": passed_sessions,
        "pass_rate": (passed_sessions / total_sessions * 100) if total_sessions > 0 else 0,
        "average_success_rate": sum(s.score_percent for s in sessions) / total_sessions,
        "total_tasks_solved": total_tasks,
        "total_correct": total_correct,
        "class_success_rate": (total_correct / total_tasks * 100) if total_tasks > 0 else 0,
    }
```

### 5.3. Анализ проблемных мест

```python
def find_problematic_tasks(ks):
    """Найти задачи, с которыми ученики чаще всего ошибаются"""
    attempts = TaskAttempt.objects.filter(
        task__ks=ks
    )
    
    task_stats = {}
    for attempt in attempts:
        task_id = attempt.task_id
        if task_id not in task_stats:
            task_stats[task_id] = {"total": 0, "correct": 0}
        
        task_stats[task_id]["total"] += 1
        if attempt.is_correct:
            task_stats[task_id]["correct"] += 1
    
    # Вычисляем процент правильных для каждой задачи
    problematic = []
    for task_id, stats in task_stats.items():
        success_rate = stats["correct"] / stats["total"] * 100
        if success_rate < 50:  # Меньше 50% правильных
            task = Task.objects.get(id=task_id)
            problematic.append({
                "task": task,
                "success_rate": success_rate,
                "total_attempts": stats["total"],
            })
    
    # Сортируем по проценту правильных (от меньшего к большему)
    problematic.sort(key=lambda x: x["success_rate"])
    
    return problematic
```

```jsx
function StageProgress({ currentStage }) {
  const stages = [
    { key: "comprehension", label: "Осмысление", icon: "📖" },
    { key: "typical_task", label: "Типовая задача", icon: "🎯" },
    { key: "task_list", label: "Задачи", icon: "📋" },
    { key: "solving", label: "Решение", icon: "✏️" },
    { key: "completed", label: "Готово", icon: "🎉" },
  ];
  
  const currentIndex = stages.findIndex(s => 
    currentStage?.startsWith(s.key) || currentStage === s.key
  );
  
  return (
    <div className="space-y-2">
      {stages.map((stage, idx) => {
        const isCompleted = idx < currentIndex;
        const isCurrent = idx === currentIndex;
        
        return (
          <div
            key={stage.key}
            className={`flex items-center gap-3 px-3 py-2 rounded-lg ${
              isCurrent ? "bg-blue-600 text-white" :
              isCompleted ? "bg-emerald-100 text-emerald-700" :
              "bg-slate-100 text-slate-500"
            }`}
          >
            <div className="w-8 h-8 rounded-full flex items-center justify-center">
              {isCompleted ? "✓" : stage.icon}
            </div>
            <span>{stage.label}</span>
          </div>
        );
      })}
    </div>
  );
}
```

## Практические рекомендации

### 1. Начните с простого

Не пытайтесь сразу реализовать сложный алгоритм. Начните с:
- Выбора ученика
- Отслеживания ошибок подряд
- Простой корректировки

### 2. Собирайте данные

Логируйте все события:
- Ответы на задачи
- Переходы между этапами
- Выборы сложности

Это поможет улучшить алгоритм.

### 3. Тестируйте на реальных данных

- Запустите систему с небольшой группой учеников
- Соберите метрики
- Проанализируйте результаты
- Скорректируйте алгоритм

### 4. Визуализируйте прогресс

Ученики должны видеть свой прогресс:
- Сколько задач решено
- Процент правильных ответов
- Текущий этап
- Достижения

### 5. Балансируйте сложность

- Не делайте слишком резкие переходы
- Давайте ученику время адаптироваться
- Учитывайте не только правильность, но и время решения
