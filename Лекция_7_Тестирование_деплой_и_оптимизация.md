## 1. Введение: зачем нужно тестирование

### 1.1. Проблема без тестирования

**Без тестирования:**
- ❌ Ошибки обнаруживаются только пользователями
- ❌ Сложно проверить, что изменения не сломали существующий функционал
- ❌ Нет уверенности в работоспособности кода
- ❌ Сложно рефакторить код

**Пример:**
```python
# Изменили функцию проверки ответа
def check_answer(self, answer):
    # Добавили новую логику
    return answer == self.correct_answer

# Как проверить, что всё работает?
# Придётся вручную тестировать в браузере...
```

### 1.2. Решение: автоматическое тестирование

**Автоматические тесты:**
- ✅ Проверяют код автоматически
- ✅ Можно запускать при каждом изменении
- ✅ Дают уверенность в работоспособности
- ✅ Упрощают рефакторинг

**Пример:**
```python
def test_check_answer():
    task = Task(correct_answer=5.0)
    assert task.check_answer(5.0) == True
    assert task.check_answer(4.0) == False
```

### 1.3. Типы тестов

**1. Unit-тесты (модульные):**
- Тестируют отдельные функции/методы
- Быстрые, изолированные
- Пример: тест метода `check_answer()`

**2. Integration-тесты (интеграционные):**
- Тестируют взаимодействие компонентов
- Проверяют работу API, БД
- Пример: тест создания курса через API

**3. End-to-end тесты (E2E):**
- Тестируют весь поток от начала до конца
- Медленные, но полные
- Пример: тест регистрации → входа → создания курса

## 2. Тестирование в Django

### 2.1. Структура тестов

**Файл `tests.py`:**
```python
from django.test import TestCase
from django.contrib.auth.models import User
from .models import Course, Task

class CourseModelTest(TestCase):
    """Тесты для модели Course"""
    
    def setUp(self):
        """Настройка перед каждым тестом"""
        self.user = User.objects.create_user(
            username='testuser',
            password='testpass123'
        )
        self.course = Course.objects.create(
            title='Тестовый курс',
            description='Описание'
        )
    
    def test_course_creation(self):
        """Тест создания курса"""
        self.assertEqual(self.course.title, 'Тестовый курс')
        self.assertIsNotNone(self.course.created_at)
    
    def test_course_str(self):
        """Тест метода __str__"""
        self.assertEqual(str(self.course), 'Тестовый курс')
```

### 2.3. TestCase vs TransactionTestCase

**TestCase** (быстрый):
- Использует транзакции для изоляции
- Каждый тест в отдельной транзакции
- Автоматически откатывает изменения

```python
class CourseModelTest(TestCase):
    def test_course_creation(self):
        course = Course.objects.create(title='Курс')
        self.assertEqual(Course.objects.count(), 1)
```

**TransactionTestCase** (медленный):
- Используется, когда нужно тестировать транзакции
- Для тестирования сигналов, которые используют БД

## 3. Тестирование API

### 3.1. APIClient

**Django REST Framework** предоставляет `APIClient` для тестирования API:

```python
from rest_framework.test import APIClient
from rest_framework import status

class CourseAPITest(TestCase):
    def setUp(self):
        self.client = APIClient()
        self.user = User.objects.create_user(
            username='testuser',
            password='testpass123'
        )
        self.client.force_authenticate(user=self.user)
    
    def test_list_courses(self):
        """Тест получения списка курсов"""
        Course.objects.create(title='Курс 1')
        Course.objects.create(title='Курс 2')
        
        response = self.client.get('/api/courses/')
        
        self.assertEqual(response.status_code, status.HTTP_200_OK)
        self.assertEqual(len(response.data), 2)
    
    def test_create_course(self):
        """Тест создания курса"""
        data = {
            'title': 'Новый курс',
            'description': 'Описание'
        }
        
        response = self.client.post('/api/courses/', data, format='json')
        
        self.assertEqual(response.status_code, status.HTTP_201_CREATED)
        self.assertEqual(Course.objects.count(), 1)
        self.assertEqual(Course.objects.get().title, 'Новый курс')
    
    def test_create_course_validation(self):
        """Тест валидации при создании"""
        data = {
            'title': '',  # Пустое название
            'description': 'Описание'
        }
        
        response = self.client.post('/api/courses/', data, format='json')
        
        self.assertEqual(response.status_code, status.HTTP_400_BAD_REQUEST)
        self.assertIn('title', response.data)
```

## 6. Оптимизация производительности

### 6.1. Оптимизация запросов к БД

#### 6.1.1. select_related (для ForeignKey)

**Проблема (N+1 запросов):**
```python
courses = Course.objects.all()
for course in courses:
    print(course.author.username)  # Отдельный запрос для каждого курса!
```

**Решение:**
```python
courses = Course.objects.select_related('author').all()
for course in courses:
    print(course.author.username)  # Данные уже загружены
```

#### 6.1.2. prefetch_related (для ManyToMany, обратные ForeignKey)

**Проблема:**
```python
courses = Course.objects.all()
for course in courses:
    print(course.modules.all())  # Отдельный запрос для каждого курса!
```

**Решение:**
```python
courses = Course.objects.prefetch_related('modules').all()
for course in courses:
    print(course.modules.all())  # Данные уже загружены
```

#### 6.1.3. only() и defer()

**Загрузка только нужных полей:**
```python
# Загрузить только title и description
courses = Course.objects.only('title', 'description').all()

# Загрузить всё кроме description (если описание большое)
courses = Course.objects.defer('description').all()
```

### 6.2. Кэширование

#### 6.2.1. Кэширование в Django

**Настройка (settings.py):**
```python
CACHES = {
    'default': {
        'BACKEND': 'django.core.cache.backends.redis.RedisCache',
        'LOCATION': 'redis://127.0.0.1:6379/1',
    }
}
```

**Использование:**
```python
from django.core.cache import cache

def get_courses():
    # Проверяем кэш
    courses = cache.get('courses_list')
    if courses is None:
        # Загружаем из БД
        courses = list(Course.objects.all())
        # Сохраняем в кэш на 1 час
        cache.set('courses_list', courses, 3600)
    return courses
```

#### 6.2.2. Кэширование в ViewSet

```python
from django.views.decorators.cache import cache_page
from django.utils.decorators import method_decorator

@method_decorator(cache_page(60 * 15), name='list')  # Кэш на 15 минут
class CourseViewSet(viewsets.ModelViewSet):
    queryset = Course.objects.all()
    serializer_class = CourseSerializer
```

## Практические рекомендации

### 1. Тестирование

- **Пишите тесты параллельно с кодом** — не откладывайте на потом
- **Тестируйте критичные части** — API, бизнес-логику, модели
- **Используйте фикстуры** — для тестовых данных
- **Запускайте тесты перед коммитом** — `git pre-commit hook`

### 2. Деплой

- **Используйте CI/CD** — автоматический деплой при push
- **Делайте резервные копии БД** — перед каждым деплоем
- **Тестируйте на staging** — перед production
- **Используйте переменные окружения** — для секретов

### 3. Оптимизация

- **Профилируйте код** — найдите узкие места
- **Используйте Django Debug Toolbar** — для разработки
- **Мониторьте производительность** — логи, метрики
- **Оптимизируйте постепенно** — не оптимизируйте преждевременно
