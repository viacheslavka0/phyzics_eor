## 1. ВВЕДЕНИЕ

Современное образование активно использует электронные образовательные ресурсы, которые позволяют студентам изучать материал в удобном темпе и получать практический опыт. Разработка качественного ЭОР требует сочетания технических навыков программирования и педагогических компетенций для структурирования материала.

Целью учебной практики являлась разработка образовательного контента для ЭОР по созданию веб-приложений с использованием Django и React. ЭОР разрабатывается в рамках магистерской диссертации и предназначен для обучения студентов самостоятельной разработке образовательных систем.

В ходе практики решались следующие задачи: разработка структуры курса, создание лекционных материалов по всем аспектам разработки, разработка практических заданий, подготовка вопросов для итогового контроля и обеспечение логической последовательности материала.

## 2. ОСНОВНАЯ ЧАСТЬ

### 2.1. Структура разработанного курса

Разработан курс "Разработка веб-приложений для образовательных систем", состоящий из 7 лекций и 6 практических заданий. Курс построен по принципу "от простого к сложному" и охватывает полный цикл разработки веб-приложения.

Структура курса обеспечивает последовательное усвоение материала. Первые лекции посвящены основам Django ORM и проектированию базы данных. Затем студенты изучают создание REST API с использованием Django REST Framework. Следующий этап — изучение React для создания пользовательского интерфейса. Далее рассматривается интеграция frontend и backend компонентов. Завершающие лекции посвящены адаптивным алгоритмам обучения, тестированию и развертыванию приложений.

### 2.2. Разработка лекционных материалов

Каждая лекция имеет единообразную структуру: план с указанием времени (общая продолжительность 90 минут), теоретический материал с примерами кода, вопросы для самопроверки и дополнительные ресурсы.

**Лекция "Модели данных и проектирование БД"** знакомит с Django ORM — технологией работы с базой данных через объекты Python. Рассматриваются типы полей (CharField, TextField, IntegerField, DateTimeField, JSONField и др.), три типа связей между моделями (ForeignKey, OneToOneField, ManyToManyField), мета-классы для настройки моделей, методы моделей и работа с миграциями. Все примеры основаны на реальном проекте EORA.

**Лекция "API и Django REST Framework"** посвящена созданию REST API. Изучаются принципы REST архитектуры, HTTP методы, формат JSON. Подробно рассматриваются сериализаторы (базовые, вложенные, с SerializerMethodField), ViewSets (ModelViewSet, ReadOnlyModelViewSet, GenericViewSet), маршрутизация с помощью Routers, кастомные действия через декоратор @action, аутентификация и обработка ошибок.

**Лекция "Frontend — React основы"** знакомит с библиотекой React. Рассматриваются основные принципы: компонентный подход, декларативность, виртуальный DOM. Изучается структура React-приложения, синтаксис JSX, состояние компонентов и хуки (useState, useEffect, useCallback, useRef), работа с API, Context API для управления состоянием, оптимизация производительности и стилизация с Tailwind CSS.

**Лекция "Интеграция Frontend и Backend"** решает задачи взаимодействия между клиентом и сервером. Рассматриваются проблемы интеграции, настройка CORS с помощью django-cors-headers, аутентификация через сессии, работа с CSRF токенами, обработка ошибок и валидации, синхронизация состояния, работа с файлами через FormData и оптимизация запросов.

**Лекция "Адаптивный алгоритм обучения"** посвящена реализации механизма адаптации сложности заданий. Изучаются модели для отслеживания прогресса (LearningSession, TaskAttempt, EventLog), различные алгоритмы адаптивного выбора заданий (на основе выбора ученика, последовательности ошибок, алгоритм "лестницы", на основе процента правильных ответов), аналитика и метрики, визуализация прогресса.

**Лекция "Тестирование, деплой и оптимизация"** завершает курс. Рассматриваются типы тестов (unit, integration, end-to-end), написание тестов в Django и для API, подготовка к production (разделение настроек, переменные окружения), развертывание на VPS и PaaS платформах, оптимизация производительности (select_related, prefetch_related, индексы, кэширование) и мониторинг.

Все лекции разработаны с практической направленностью — примеры кода основаны на реальном проекте EORA. Материал излагается пошагово, от простого к сложному, каждая лекция самодостаточна.

### 2.3. Разработка практических заданий

Практические задания имеют единообразную структуру: цель задания, предварительные требования, детальные требования с пошаговыми инструкциями, критерии оценивания (25 баллов), пример минимального решения, рекомендации и FAQ.

**Задание "Модели данных"** требует создания минимум 4 связанных Django моделей с различными типами полей, реализацией всех типов связей, созданием Meta классов и методов моделей, применением миграций.

**Задание "API и Django REST Framework"** направлено на освоение создания REST API. Требуется настроить DRF, создать минимум 3 сериализатора (базовый, вложенный, с SerializerMethodField), реализовать 2 ViewSet, настроить маршрутизацию, создать кастомные действия и реализовать валидацию.

**Задание "Frontend React"** требует создания React-приложения с Vite и Tailwind CSS. Нужно создать компонентную структуру, реализовать минимум 4 компонента с различными хуками, организовать взаимодействие с API с обработкой ошибок, реализовать обработку событий и стилизацию.

**Задание "Интеграция Frontend и Backend"** объединяет предыдущие знания. Требуется настроить CORS, создать утилиты для работы с API, реализовать аутентификацию, организовать обработку ошибок, синхронизировать состояние и работать с файлами через FormData.

**Задание "Адаптивный алгоритм обучения"** посвящено реализации адаптивного алгоритма. Нужно создать модели для отслеживания прогресса, реализовать минимум один алгоритм выбора заданий, создать API эндпоинты, визуализировать прогресс и протестировать алгоритм.

**Задание "Тестирование, деплой и оптимизация"** завершает цикл. Требуется написать минимум 5 тестов для моделей и 5 тестов для API, подготовить проект к production, оптимизировать запросы к БД и создать документацию по развертыванию.

Система оценивания прозрачна: каждое задание оценивается по 25-балльной шкале с детальными критериями по технической реализации, качеству кода, документации и дополнительным заданиям.

### 2.4. Разработка вопросов для итогового контроля

Разработан финальный тест из 30 вопросов в формате GIFT для Moodle. Вопросы проверяют как теоретические знания, так и понимание практических аспектов.

Распределение вопросов: 4 вопроса по архитектуре, 5 по моделям данных, 5 по API и DRF, 5 по React, 4 по интеграции, 4 по адаптивным алгоритмам, 3 по тестированию и деплою. Все вопросы имеют формат выбора одного ответа из четырех вариантов. Вопросы сформулированы для проверки не только знаний фактов, но и понимания концепций и их применения.

### 2.5. Использованные технологии

В процессе разработки использовались: Markdown для написания лекций и заданий, формат GIFT для создания вопросов в Moodle, Git для версионирования материалов, реальный проект EORA как источник примеров кода.
