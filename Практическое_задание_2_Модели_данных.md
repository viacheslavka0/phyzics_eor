## Задание

Создайте Django-проект с моделями данных для вашего будущего ЭОР (по вашей предметной области). Проект должен содержать минимум **4 связанные модели**, которые отражают структуру учебного контента.

### Требования к моделям

#### 1. Обязательные модели (минимум 4)

Создайте иерархию моделей, например:

**Вариант А (для большинства предметов):**
- `Course` (Курс/Дисциплина) — верхний уровень
- `Module` (Модуль/Раздел) — связан с Course
- `Lesson` (Урок/Тема) — связан с Module
- `Question` (Вопрос/Задание) — связан с Lesson

**Вариант Б (для предметов с практикой):**
- `Subject` (Предмет)
- `Topic` (Тема)
- `Exercise` (Упражнение) — связан с Topic
- `Attempt` (Попытка решения) — связан с Exercise

**Вариант В (для вашей предметной области):**
- Придумайте свою структуру, но она должна быть логичной и отражать реальную иерархию учебного материала.

#### 2. Требования к полям моделей

Каждая модель должна содержать:

**a) Базовые поля:**
- Минимум одно текстовое поле (`CharField` или `TextField`)
- Поле для порядка/сортировки (`PositiveIntegerField` с `default=1`)
- Поля даты/времени (`DateTimeField` с `auto_now_add=True` и `auto_now=True`)

**b) Связи:**
- Используйте минимум **3 типа связей**:
  - `ForeignKey` (один-ко-многим) — минимум 2 связи
  - `OneToOneField` (один-к-одному) — минимум 1 связь
  - `ManyToManyField` (многие-ко-многим) — опционально, но желательно

**c) Специальные поля:**
- Поле с выбором (`choices`) — минимум в одной модели
- `BooleanField` — минимум в одной модели
- `ImageField` или `FileField` — минимум в одной модели
- `JSONField` — **обязательно** в одной из моделей (для гибких данных)

**d) Мета-класс:**
- В каждой модели должен быть класс `Meta` с:
  - `verbose_name` и `verbose_name_plural`
  - `ordering` (сортировка по умолчанию)

**e) Метод `__str__()`:**
- Каждая модель должна иметь метод `__str__()` для читаемого отображения

#### 3. Методы моделей

Добавьте минимум **2 метода** в разные модели:

**a) Метод для бизнес-логики:**
- Например: проверка ответа, вычисление балла, валидация данных

**b) Метод для работы с JSONField:**
- Если используете `JSONField`, добавьте метод для работы с ним (например, `get_all_options()`, `check_answers()`)

#### 4. Пример использования JSONField

В одной из моделей используйте `JSONField` для хранения гибких данных. Например:

- Варианты ответов в вопросах
- Настройки упражнения
- Метаданные урока
- Структурированные данные

**Пример структуры:**
```python
class Question(models.Model):
    text = models.TextField()
    # Варианты ответов в JSON
    options = models.JSONField(
        default=list,
        help_text='[{"text": "Вариант 1", "is_correct": true}, ...]'
    )
    
    def check_answer(self, answer_index):
        """Проверка ответа ученика"""
        if 0 <= answer_index < len(self.options):
            return self.options[answer_index].get("is_correct", False)
        return False
```

## Пример минимального решения

Ниже приведён пример минимального решения для справки. **Не копируйте его полностью!** Используйте как ориентир и адаптируйте под свою предметную область.

```python
from django.db import models

# =============================================================================
# 1. УЧЕБНАЯ ИЕРАРХИЯ
# =============================================================================

class Course(models.Model):
    """Курс/Дисциплина"""
    title = models.CharField(max_length=255)
    description = models.TextField(blank=True)
    created_at = models.DateTimeField(auto_now_add=True)
    updated_at = models.DateTimeField(auto_now=True)
    
    class Meta:
        verbose_name = "Курс"
        verbose_name_plural = "Курсы"
        ordering = ["-created_at"]
    
    def __str__(self):
        return self.title


class Module(models.Model):
    """Модуль/Раздел курса"""
    course = models.ForeignKey(
        Course,
        on_delete=models.CASCADE,
        related_name="modules"
    )
    title = models.CharField(max_length=255)
    order = models.PositiveIntegerField(default=1)
    created_at = models.DateTimeField(auto_now_add=True)
    
    class Meta:
        verbose_name = "Модуль"
        verbose_name_plural = "Модули"
        ordering = ["order"]
    
    def __str__(self):
        return f"{self.course.title} - {self.title}"


class Lesson(models.Model):
    """Урок/Тема модуля"""
    STATUS_CHOICES = [
        ("draft", "Черновик"),
        ("published", "Опубликовано"),
    ]
    
    module = models.ForeignKey(
        Module,
        on_delete=models.CASCADE,
        related_name="lessons"
    )
    title = models.CharField(max_length=255)
    content = models.TextField()
    status = models.CharField(
        max_length=10,
        choices=STATUS_CHOICES,
        default="draft"
    )
    order = models.PositiveIntegerField(default=1)
    is_active = models.BooleanField(default=True)
    created_at = models.DateTimeField(auto_now_add=True)
    updated_at = models.DateTimeField(auto_now=True)
    
    class Meta:
        verbose_name = "Урок"
        verbose_name_plural = "Уроки"
        ordering = ["order"]
    
    def __str__(self):
        return f"{self.module.title} - {self.title}"


class Question(models.Model):
    """Вопрос к уроку"""
    TYPE_CHOICES = [
        ("single", "Выбор одного"),
        ("multiple", "Множественный выбор"),
        ("text", "Текстовый ответ"),
    ]
    
    lesson = models.ForeignKey(
        Lesson,
        on_delete=models.CASCADE,
        related_name="questions"
    )
    type = models.CharField(
        max_length=20,
        choices=TYPE_CHOICES,
        default="single"
    )
    text = models.TextField()
    
    # JSONField для вариантов ответов
    options = models.JSONField(
        default=list,
        blank=True,
        help_text='[{"text": "Вариант 1", "is_correct": true}, ...]'
    )
    
    correct_answer_text = models.CharField(
        max_length=500,
        blank=True,
        help_text="Правильный ответ (для текстовых вопросов)"
    )
    
    order = models.PositiveIntegerField(default=1)
    created_at = models.DateTimeField(auto_now_add=True)
    
    class Meta:
        verbose_name = "Вопрос"
        verbose_name_plural = "Вопросы"
        ordering = ["order"]
    
    def __str__(self):
        return f"Вопрос {self.order}: {self.text[:50]}..."
    
    def check_answer(self, answer):
        """Проверка ответа ученика"""
        if self.type == "text":
            return answer.strip().lower() == self.correct_answer_text.strip().lower()
        
        elif self.type == "single":
            # answer — индекс выбранного варианта
            if 0 <= answer < len(self.options):
                return self.options[answer].get("is_correct", False)
            return False
        
        elif self.type == "multiple":
            # answer — список индексов
            correct_indices = {
                i for i, opt in enumerate(self.options)
                if opt.get("is_correct")
            }
            return set(answer) == correct_indices
        
        return False
    
    def get_all_options(self):
        """Получить все варианты ответов"""
        return [opt["text"] for opt in self.options]


# OneToOneField пример (опционально)
class LessonSettings(models.Model):
    """Настройки урока (один-к-одному)"""
    lesson = models.OneToOneField(
        Lesson,
        on_delete=models.CASCADE,
        related_name="settings"
    )
    allow_comments = models.BooleanField(default=True)
    show_hints = models.BooleanField(default=False)
    max_attempts = models.PositiveIntegerField(default=3)
    
    class Meta:
        verbose_name = "Настройки урока"
        verbose_name_plural = "Настройки уроков"
    
    def __str__(self):
        return f"Настройки: {self.lesson.title}"
```

## Часто задаваемые вопросы

**Q: Можно ли использовать больше 4 моделей?**  
A: Да, можно! Это даже приветствуется, если это оправдано вашей предметной областью.

**Q: Обязательно ли использовать все типы связей?**  
A: ForeignKey и OneToOneField — обязательно. ManyToManyField — опционально, но желательно.

**Q: Что делать, если не знаю, какую структуру выбрать?**  
A: Подумайте о реальной иерархии учебного материала в вашем предмете. Например:
- Для языков: Курс → Урок → Упражнение → Вопрос
- Для программирования: Курс → Модуль → Задача → Тест
- Для истории: Предмет → Период → Событие → Вопрос

**Q: Нужно ли создавать реальные данные в БД?**  
A: Нет, достаточно создать модели и применить миграции. Создание данных — опционально.

**Q: Что делать, если миграции не применяются?**  
A: Проверьте синтаксис моделей командой `python manage.py check`. Убедитесь, что все импорты корректны.
