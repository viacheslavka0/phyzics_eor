# ВВЕДЕНИЕ

## Наименование системы

### Полное наименование системы

Адаптивная обучающая платформа EORA для изучения физики с использованием систем знаний.

### Краткое наименование системы

EORA Learning Platform.

## Область применения

АИС EORA предназначена для применения в образовательных учреждениях (школах, колледжах, университетах) для автоматизации процесса обучения физике учащихся 7-9 классов с использованием адаптивных алгоритмов и интерактивных методов обучения.

Система может быть интегрирована в существующие учебные процессы как основной или дополнительный инструмент изучения физики.

## Возможности программы

Система обеспечивает выполнение следующих основных функций:

1) **Адаптивное обучение** - система автоматически подстраивается под уровень знаний и способности ученика;

2) **Интерактивные системы знаний** - изучение физических явлений через структурированные таблицы знаний с активными зонами;

3) **Решение задач** - автоматическая проверка численных и текстовых ответов с учетом физических погрешностей;

4) **Построение схем** - интерактивный редактор для создания моделей физических ситуаций;

5) **Пооперационный контроль** - пошаговое решение задач с проверкой каждого этапа;

6) **Аналитика прогресса** - детальная статистика обучения для учеников и преподавателей;

7) **Многоуровневая структура курса** - иерархическая организация материала по классам, разделам и темам.

## Уровень подготовки пользователя

### Для учеников:

Основные требования к ученику:
1) Базовые навыки работы с компьютером и интернет-браузером;
2) Знание школьной программы по физике соответствующего класса;
3) Умение работать с математическими формулами и единицами измерения;
4) Навыки чтения и понимания физических задач.

### Для преподавателей:

Дополнительные требования к преподавателю:
1) Знание методики преподавания физики;
2) Понимание принципов адаптивного обучения;
3) Опыт работы с образовательными технологиями;
4) Навыки создания и редактирования учебного контента.

### Основные обязанности пользователя:

1) Соблюдение правил информационной безопасности при работе с системой;
2) Своевременное сообщение администратору о технических неполадках;
3) Корректное использование функционала системы согласно назначению;
4) Сохранение конфиденциальности личных данных других пользователей.

## Перечень эксплуатационной документации

Для работы в системе пользователю рекомендуется изучить следующую документацию:

1) **Руководство пользователя** (настоящий документ) - основной документ для работы с системой;
2) **Методические рекомендации** - дополнительные материалы по эффективному использованию платформы;
3) **Техническое описание** - детальная информация об архитектуре и принципах работы системы;
4) **Руководство администратора** - для технических специалистов, обслуживающих систему.

# 1 НАЗНАЧЕНИЕ И УСЛОВИЯ ПРИМЕНЕНИЯ ПРОГРАММЫ

## 1.1 Назначение программы

АИС EORA предназначена для повышения эффективности образовательного процесса по физике, персонализации обучения и автоматизации оценки знаний учащихся.

Основным назначением системы является:

- **Индивидуализация образовательного процесса** через адаптацию к способностям и темпу обучения каждого ученика;
- **Повышение качества усвоения материала** за счет интерактивных методов и немедленной обратной связи;
- **Автоматизация процесса оценивания** с объективной и точной фиксацией прогресса;
- **Оптимизация времени преподавателя** через автоматическую проверку заданий и генерацию отчетов.

## 1.2 Функции, выполняемые программой

### 1.2.1 Подсистема аутентификации и авторизации

**Функция входа в систему:**
- Аутентификация пользователей через систему логин/пароль;
- Интеграция с единой системой аутентификации образовательного учреждения;
- Разграничение прав доступа по ролям (ученик, преподаватель, администратор);
- Восстановление доступа при утере пароля.

**Управление профилем пользователя:**
- Просмотр и редактирование личных данных;
- Настройка предпочтений интерфейса;
- Просмотр статистики обучения;
- Управление уведомлениями.

### 1.2.2 Подсистема каталога курса

**Структурированное представление учебного материала:**
- Иерархическое отображение: Классы → Разделы → Темы → Системы знаний;
- Индикаторы прогресса изучения материала;
- Фильтрация и поиск по содержимому;
- Рекомендации следующих тем для изучения.

### 1.2.3 Подсистема адаптивного обучения

**Семиэтапный алгоритм обучения:**

1. **Этап "Осмысление СК"** - интерактивное изучение системы знаний с активными зонами и заданиями cloze;
2. **Этап "Типовая задача"** - понимание класса задач, решаемых данной системой знаний;
3. **Этап "Список задач"** - ознакомление с набором задач различной сложности;
4. **Этап "Оценка трудности"** - субъективная оценка сложности материала учеником;
5. **Этап "Решение задач"** - адаптивный выбор уровня сложности (легкое/среднее/сложное);
6. **Этап "Составление метода"** - создание алгоритма решения для сложных задач;
7. **Этап "Пооперационный контроль"** - пошаговое решение с проверкой каждого действия.

**Адаптивная логика:**
- Автоматический переход между уровнями сложности на основе результатов;
- Персонализированные рекомендации по дальнейшему обучению;
- Учет индивидального темпа освоения материала.

### 1.2.4 Подсистема интерактивных заданий

**Типы заданий:**
- **Соответствие зон** - клики по активным областям на изображениях систем знаний;
- **Cloze-тексты** - заполнение пропусков в тексте из банка слов;
- **Численные задачи** - ввод числовых ответов с автоматической проверкой;
- **Текстовые ответы** - открытые вопросы с проверкой по ключевым словам;
- **Множественный выбор** - выбор одного или нескольких правильных вариантов.

**Система проверки:**
- Автоматическая проверка с учетом физических погрешностей;
- Нечеткое сравнение текстовых ответов;
- Немедленная обратная связь с объяснением ошибок;
- Возможность повторных попыток с адаптацией сложности.

### 1.2.5 Подсистема редактора схем

**Построение моделей физических ситуаций:**
- Палитра физических элементов (тела, векторы, оси координат, точки);
- Инструменты рисования и редактирования;
- Привязка к сетке и объектам для точного позиционирования;
- История изменений с возможностью отмены/повтора действий;
- Экспорт схем в различные форматы.

**Проверка схем:**
- Автоматическое сравнение с эталонными решениями;
- Анализ топологии и пространственных отношений элементов;
- Подсказки и рекомендации по улучшению схемы.

### 1.2.6 Подсистема аналитики и отчетности

**Для учеников:**
- Детальная статистика прогресса по темам и разделам;
- Анализ слабых мест и рекомендации по их устранению;
- Сравнение с общими показателями группы (анонимно);
- Планирование дальнейшего обучения.

**Для преподавателей:**
- Мониторинг успеваемости класса и отдельных учеников;
- Выявление проблемных тем и концепций;
- Отчеты по эффективности различных методов обучения;
- Экспорт данных для интеграции с другими системами.

### 1.2.7 Подсистема управления контентом (для преподавателей)

**Создание и редактирование систем знаний:**
- Загрузка и разметка изображений таблиц знаний;
- Создание интерактивных зон и привязка вопросов;
- Настройка параметров проверки и пороговых значений;
- Предварительный просмотр созданного контента.

**Управление задачами:**
- Создание новых задач с различными типами ответов;
- Настройка эталонных решений и критериев оценки;
- Группировка задач по уровням сложности;
- Импорт задач из внешних источников.

## 1.3 Требования к техническому обеспечению

4. **Резервное копирование:** Автоматическая система бэкапов
5. **Мониторинг:** Система мониторинга производительности

**Минимальные требования к рабочему месту пользователя:**

2. **Оперативная память:** 4 ГБ

4. **Дисковое пространство:** 1 ГБ свободного места для кэша браузера
5. **Монитор:** Разрешение не менее 1366×768 пикселей
6. **Устройства ввода:** Клавиатура и мышь (или тачпад)
7. **Интернет-соединение:** Широкополосное соединение от 5 Мбит/с

**Поддерживаемые браузеры:**
- Google Chrome версии 90 и выше
- Mozilla Firefox версии 88 и выше  
- Microsoft Edge версии 90 и выше
- Safari версии 14 и выше (для macOS)

**Рекомендуемые параметры:**
1. **Процессор:** Intel Core i5-10400 / AMD Ryzen 5 3600 или выше
2. **Оперативная память:** 8 ГБ или больше
3. **Монитор:** Full HD (1920×1080) или выше для комфортной работы
4. **Интернет:** Соединение от 10 Мбит/с для стабильной работы

### Дополнительные требования

**Программное обеспечение:**
- Включенный JavaScript в браузере
- Поддержка HTML5 Canvas для редактора схем
- Включенные cookies для сохранения сессии
- Отключенная блокировка всплывающих окон для домена системы

**Безопасность:**
- Актуальные обновления операционной системы
- Обновленный антивирус с исключениями для браузера
- Корректно настроенный системный брандмауэр

# 2 ПОДГОТОВКА К РАБОТЕ

## 2.1 Состав и содержание дистрибутивного носителя данных

Дистрибутив системы EORA поставляется в виде Docker-контейнеров и включает следующие компоненты:

**Основные файлы системы:**
- `eora-backend.tar` - Docker-образ серверной части (Django)
- `eora-frontend.tar` - Docker-образ клиентской части (React)
- `eora-database.tar` - Docker-образ базы данных (PostgreSQL)
- `eora-redis.tar` - Docker-образ кэш-сервера (Redis)

**Конфигурационные файлы:**
- `docker-compose.yml` - основной файл конфигурации Docker Compose
- `nginx.conf` - конфигурация веб-сервера
- `.env.example` - шаблон файла переменных окружения
- `init.sql` - скрипт инициализации базы данных

**Документация:**
- `README.md` - краткое руководство по установке
- `INSTALL.md` - детальная инструкция по развертыванию
- `backup.sh` - скрипт создания резервных копий
- `update.sh` - скрипт обновления системы

**Демонстрационные данные:**
- `demo_data.sql` - демонстрационный курс физики
- `sample_images/` - примеры изображений систем знаний
- `test_users.csv` - тестовые учетные записи пользователей

## 2.2 Установка и загрузка программы

### 2.2.1 Подготовка сервера

**Шаг 1. Установка Docker и Docker Compose:**

```bash
# Обновление системы
sudo apt update && sudo apt upgrade -y

# Установка Docker
curl -fsSL https://get.docker.com -o get-docker.sh
sh get-docker.sh

# Установка Docker Compose
sudo apt install docker-compose -y

# Добавление пользователя в группу docker
sudo usermod -aG docker $USER
```

**Шаг 2. Создание рабочей директории:**

```bash
# Создание директории для системы
sudo mkdir -p /opt/eora
sudo chown $USER:$USER /opt/eora
cd /opt/eora
```

**Шаг 3. Распаковка дистрибутива:**

```bash
# Распаковка архива с системой
tar -xzf eora-distribution.tar.gz

# Загрузка Docker-образов
docker load -i eora-backend.tar
docker load -i eora-frontend.tar
docker load -i eora-database.tar
docker load -i eora-redis.tar
```

### 2.2.2 Конфигурация системы

**Шаг 1. Настройка переменных окружения:**

```bash
# Копирование шаблона конфигурации
cp .env.example .env

# Редактирование конфигурации
nano .env
```

**Основные параметры в файле .env:**
```ini
# База данных
POSTGRES_DB=eora_db
POSTGRES_USER=eora_user
POSTGRES_PASSWORD=secure_password_here

# Django
SECRET_KEY=your_secret_key_here
DEBUG=False
ALLOWED_HOSTS=your-domain.com,localhost

# Внешние сервисы
REDIS_URL=redis://redis:6379/1
EMAIL_HOST=smtp.your-domain.com
EMAIL_PORT=587
EMAIL_USE_TLS=True
```

**Шаг 2. Генерация SSL-сертификатов (для продакшена):**

```bash
# Использование Let's Encrypt
sudo apt install certbot -y
certbot certonly --standalone -d your-domain.com
```

### 2.2.3 Запуск системы

**Шаг 1. Инициализация базы данных:**

```bash
# Запуск только базы данных
docker-compose up -d database

# Ожидание готовности базы данных
sleep 30

# Применение миграций
docker-compose run --rm backend python manage.py migrate

# Загрузка демонстрационных данных (опционально)
docker-compose run --rm backend python manage.py loaddata demo_data.json
```

**Шаг 2. Создание суперпользователя:**

```bash
# Создание администратора системы
docker-compose run --rm backend python manage.py createsuperuser
```

**Шаг 3. Полный запуск системы:**

```bash
# Запуск всех сервисов
docker-compose up -d

# Проверка статуса
docker-compose ps
```

## 2.3 Проверка работоспособности

### 2.3.1 Проверка доступности сервисов

**Проверка веб-интерфейса:**
1. Откройте браузер и перейдите по адресу `http://your-server-ip/`
2. Должна отобразиться главная страница EORA с формой входа
3. Проверьте отсутствие ошибок в консоли браузера (F12)

**Проверка административной панели:**
1. Перейдите по адресу `http://your-server-ip/admin/`
2. Войдите с учетными данными суперпользователя
3. Убедитесь в корректном отображении разделов управления

**Проверка API:**
1. Откройте адрес `http://your-server-ip/api/catalog/`
2. Должен отобразиться JSON с каталогом курса
3. Статус ответа должен быть 401 (требуется авторизация)

### 2.3.2 Функциональное тестирование

**Создание тестового пользователя:**
1. Войдите в админ-панель как суперпользователь
2. Перейдите в раздел "Пользователи" → "Добавить"
3. Создайте тестового ученика с логином `test_student`

- Затем решение задач с возможностью обращения к методу

- Развернутая обратная связь

- Система учитывает физически обоснованную погрешность (обычно ±2%)

# 4 АВАРИЙНЫЕ СИТУАЦИИ

## 4.1 Ошибки входа в систему

### 4.1.1 Неверные учетные данные

**Симптом:** При попытке входа отображается сообщение "Неверный логин или пароль".

**Возможные причины:**
1. Ошибка в написании логина или пароля
2. Включен режим Caps Lock
3. Неправильная раскладка клавиатуры  
4. Учетная запись заблокирована администратором

**Решение:**
1. Проверьте правильность ввода логина и пароля
2. Убедитесь, что Caps Lock отключен
3. Проверьте текущую раскладку клавиатуры (рус/eng)
4. Воспользуйтесь функцией "Показать пароль" для проверки правильности ввода
5. При повторении ошибки обратитесь к администратору системы

### 4.1.2 Ошибка соединения с сервером

**Симптом:** Сообщение "Не удается подключиться к серверу" или долгая загрузка страницы входа.

**Решение:**
1. Проверьте соединение с Интернетом
2. Убедитесь в правильности адреса системы
3. Попробуйте обновить страницу (F5)
4. Очистите кэш браузера (Ctrl+Shift+Delete)
5. Попробуйте использовать другой браузер
6. Обратитесь к системному администратору при сохранении проблемы

### 4.1.3 Заблокированная учетная запись

**Симптом:** Сообщение "Учетная запись временно заблокирована".

**Причина:** Превышено количество неудачных попыток входа (обычно 5 попыток).

**Решение:**
1. Подождите 15-30 минут для автоматической разблокировки
2. Обратитесь к преподавателю или администратору для ручной разблокировки
3. При разблокировке смените пароль на новый

## 4.2 Ошибки при работе с задачами

### 4.2.1 Ошибка "Неверный формат ответа"

**Симптом:** При отправке численного ответа система сообщает о неверном формате.

**Возможные причины:**
1. Использование запятой вместо точки как десятичного разделителя
2. Ввод текста в поле для числа
3. Использование научной нотации в неподходящем контексте

**Решение:**
1. Используйте точку (.) как десятичный разделитель: `3.14` вместо `3,14`
2. Вводите только цифры и десятичную точку: `9.8` вместо `g=9.8 м/с²`
3. Единицы измерения выбирайте из выпадающего списка, не вводите в числовое поле
4. Для очень больших или малых чисел используйте обычную запись: `300000000` вместо `3e8`

### 4.2.2 Ответ не засчитывается как правильный

**Симптом:** Введенный правильный ответ система оценивает как неверный.

**Возможные причины:**
1. Превышена допустимая погрешность
2. Неправильно выбраны единицы измерения
3. Недостаточная или избыточная точность

**Решение:**
1. Проверьте единицы измерения - они должны соответствовать требуемым в задаче
2. Округлите ответ до разумного количества значащих цифр (обычно 2-3)
3. Убедитесь в правильности промежуточных вычислений
4. При сохранении проблемы используйте кнопку "Сообщить об ошибке"

### 4.2.3 Зависание при загрузке задачи

**Симптом:** Задача загружается очень долго или не загружается совсем.

**Решение:**
1. Обновите страницу (F5)
2. Проверьте стабильность интернет-соединения
3. Закройте другие вкладки браузера для освобождения памяти
4. Попробуйте открыть задачу в режиме инкогнито
5. При повторении проблемы сообщите администратору

## 4.3 Проблемы с загрузкой контента

### 4.3.1 Изображения не отображаются

**Симптом:** Вместо изображений систем знаний отображаются пустые области или значки ошибки.

**Решение:**
1. Проверьте скорость интернет-соединения
2. Отключите блокировщики рекламы для домена системы
3. Разрешите загрузку изображений в настройках браузера
4. Очистите кэш браузера и перезагрузите страницу
5. Попробуйте использовать другой браузер

### 4.3.2 Редактор схем не загружается

**Симптом:** При попытке открыть редактор схем появляется сообщение об ошибке или пустой экран.

**Причины:**
1. Отключен JavaScript в браузере
2. Устаревшая версия браузера
3. Блокировка WebGL видеодрайвером

**Решение:**
1. Убедитесь, что JavaScript включен в настройках браузера
2. Обновите браузер до последней версии
3. Обновите драйверы видеокарты
4. Попробуйте отключить аппаратное ускорение в браузере
5. Используйте альтернативный браузер

### 4.3.3 Медленная работа интерфейса

**Симптом:** Интерфейс системы медленно реагирует на действия пользователя.

**Решение:**
1. Закройте неиспользуемые вкладки и приложения
2. Перезапустите браузер
3. Проверьте использование оперативной памяти (должно быть свободно не менее 2 ГБ)
4. Временно отключите расширения браузера
5. Используйте браузер Google Chrome или Mozilla Firefox последних версий

## 4.4 Ошибки в редакторе схем

### 4.4.1 Элементы не перетаскиваются

**Симптом:** Невозможно перемещать элементы на холсте редактора.

**Решение:**
1. Убедитесь, что элемент не зафиксирован (нет значка замка)
2. Проверьте, что не активен режим только для чтения
3. Нажмите левой кнопкой мыши для выделения элемента перед перемещением
4. Обновите страницу и попробуйте снова

### 4.4.2 Схема не сохраняется

**Симптом:** После создания схемы она не сохраняется или пропадает при перезагрузке.

**Решение:**
1. Явно сохраните схему кнопкой "Сохранить"
2. Проверьте стабильность интернет-соединения
3. Убедитесь, что не превышен лимит времени сессии
4. Экспортируйте схему как изображение для резервной копии

### 4.4.3 Ошибка экспорта схемы

**Симптом:** При попытке экспорта схемы появляется ошибка или загружается поврежденный файл.

**Решение:**
1. Попробуйте экспорт в другом формате (PNG вместо SVG)
2. Упростите схему, удалив лишние элементы
3. Используйте функцию "Копировать в буфер обмена"
4. Сделайте скриншот как альтернативный способ сохранения

### 4.4.4 Некорректное отображение элементов

**Симптом:** Элементы схемы отображаются искаженно или в неправильных цветах.

**Решение:**
1. Обновите драйверы видеокарты
2. Отключите аппаратное ускорение в браузере
3. Измените масштаб страницы на 100% (Ctrl+0)
4. Попробуйте использовать другой браузер
5. Перезагрузите компьютер для сброса графических настроек

| Физическая величина | v₁ | Обозначение с индексами | Символ, индекс, размер |

### Рекомендации по использованию элементов

**Для задач кинематики:**
- Используйте тела-точки для материальных точек
- Размещайте координатные оси в начале движения  
- Векторы скорости и ускорения должны соответствовать направлению движения
- Размерные линии помогают указать пройденное расстояние

**Для задач динамики:**
- Векторы сил должны исходить из точки приложения
- Используйте разные цвета для различных типов сил
- Координатные оси выбирайте для упрощения проекций сил
- Подписывайте все векторы соответствующими обозначениями

**Общие принципы построения схем:**
1. Начинайте с размещения основных объектов (тел)
2. Добавляйте координатную систему
3. Размещайте векторы согласно физическому смыслу
4. Подписывайте все важные элементы
5. Используйте размерные линии для указания расстояний
6. Проверяйте соответствие схемы условию задачи
