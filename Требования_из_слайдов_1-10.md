# Требования к реализации на основе слайдов 1-10 сценария

## 📋 Что нужно реализовать согласно слайдам 1-10

### Слайд 1: Титульная страница
- ✅ Уже реализовано: Главная страница с приветствием

---

### Слайд 2: Выбор раздела и темы
**Требования:**
- Показать меню с разделами ШКФ (школьного курса физики)
- Показать темы в разделах
- Ученик выбирает раздел и тему

**Статус:** ✅ Уже реализовано в каталоге курса

---

### Слайд 3: Выбор элемента или системы знаний
**Требования:**
- Показать список элементов и систем знаний по выбранной теме
- Ученик выбирает элемент или систему знаний

**Статус:** ✅ Уже реализовано в каталоге курса

---

### Слайд 4: Осмысление СК (Расшифровка таблицы)
**Требования:**
- Показать таблицу-систему знаний с символами
- Показать описание таблицы
- Задания для расшифровки:
  1. Выдели на модели...
  2. Выдели уравнения зависимости...
  3. Выдели графики зависимости...
- Кнопка "Отправить ответ"
- Кнопка "Продолжить"

**Статус:** ✅ Частично реализовано
- Есть интерактивные зоны и вопросы
- Есть cloze-задания
- Нужно проверить соответствие заданий из слайда

---

### Слайд 5-6: Формулировка типовой задачи
**Требования:**
- Показать описание: "Модели равномерного и неравномерного движения... описывают множество конкретных ситуаций"
- Вопрос: "С какой целью и в каких ситуациях можно применить эту систему знаний?"
- Задание: "Сформулируй в общем виде задачу, которую можно решить в разнообразных ситуациях"
- Поле для ввода формулировки задачи
- Кнопка "Отправить ответ"
- После отправки показать эталонную формулировку: "Описать движение и найти величины, характеризующие равномерное или неравномерное движение конкретных тел"
- Кнопка "Пояснить задачу"
- Кнопка "Продолжить"

**Статус:** ❌ НЕ реализовано
**Что нужно сделать:**
1. Добавить поле `typical_task_formulation` в модель `LearningSession`
2. Создать UI для ввода формулировки типовой задачи
3. Показывать эталонную формулировку после ввода (самопроверка)
4. API endpoint для сохранения формулировки

---

### Слайд 7: Задание
**Требования:**
- Показать полный текст задания
- Кнопка "Продолжить"

**Статус:** ✅ Частично реализовано
- Есть этап `task_list` - список задач
- Нужно проверить, что показывается полный текст задания

---

### Слайд 8: Оценка степени трудности задания
**Требования:**
- Вопрос: "Оцени степень трудности задания и выбери порядок дальнейшей работы"
- Три варианта выбора:
  1. "Попробую решить самостоятельно" → Вариант 1 (легкое)
  2. "Хорошо бы сначала разобрать пример" → Вариант 2 (непростое)
  3. "Нужно обсудить способ решения и разобрать пример" → Вариант 3 (трудное)

**Статус:** ⚠️ Частично реализовано
**Что нужно сделать:**
1. Изменить UI этапа `difficulty_assessment`
2. Показать текстовые варианты вместо простых кнопок easy/medium/hard
3. Сохранить выбор в `difficulty_choice`

---

### Слайд 9: Решение первой задачи (Вариант 1 - легкое)
**Требования:**
- Показать условие: "Реши первую задачу"
- Требования к решению:
  - Схематический рисунок ситуации задачи как на физической модели явления
  - Формулы
  - Расчет
  - Ответ
- Поле для ввода ответа: "Ответ. ____________________________"
- Кнопка "Отправить ответ"
- Если ответ верный → переход к слайду 10
- Если ответ неверный → показать правильный ответ и предложить решить вторую задачу

**Статус:** ✅ Частично реализовано
**Что нужно доработать:**
1. Убедиться, что показывается требование о схеме, формулах, расчете
2. После неверного ответа показывать правильный ответ
3. При двух неверных решениях → переход к варианту 2 (непростое)

---

### Слайд 10: Решение второй задачи (если первая решена верно)
**Требования:**
- Показать: "Твой ответ правильный"
- Показать: "Реши вторую задачу"
- Те же требования к решению (схема, формулы, расчет, ответ)
- Поле для ввода ответа
- Кнопка "Отправить ответ"
- После решения показать правильный ответ
- Показать правильное развернутое решение
- Предложение: "Проверь свое решение по приведенному правильному развернутому решению и оцени по критериям"
- Показать вариант решения или загрузка файла с решением
- Сравнение: показать правильный ответ/фрагменты правильного решения (расчетная формула и ответ)
- Предложение оценить трудность задания/задач типа... (легкое, непростое, трудное)
- Подтверждение: "Система констатирует, что задание 1 на первый взгляд кажется легким/непростым/сложным"
- Кнопка "Продолжить" / "Не сейчас"

**Статус:** ⚠️ Частично реализовано
**Что нужно доработать:**
1. Показывать развернутое решение после каждой задачи
2. Явное предложение сравнить свое решение с эталонным
3. Показывать обратную связь: "задание на первый взгляд кажется [легким/непростым/сложным]"
4. Предложение оценить трудность (но это не оценка каждой задачи, а общая оценка типа задач)
